# 开发说明

## 项目目的

这个项目是一个手机优先的酒精计算工具，主要服务于以下场景：

- 蒸馏后样品，用酒精计读数和温度换算到 `20℃` 标准酒精度
- 已知酒度和重量或体积，进行单位换算
- 多种不同酒度的酒混合后，计算最终酒度
- 已知目标酒度和目标数量，反推两种原酒的使用量
- 高度酒加水降到目标酒度，计算需要加多少水

## 当前页面结构

- `index.html`
  - 功能首页，只负责导航
- `temp.html`
  - 温度换算页面
  - 这是当前最优先、最核心的页面
  - 页面内置独立计算逻辑，不依赖其他前端页面脚本
- `convert.html`
  - 重量体积换算页面
- `blend.html`
  - 混合结果页面
- `target.html`
  - 目标反推页面
- `dilute.html`
  - 加水降度页面

## 数据来源

- `data.js`
  - 包含 `GB 5009.225-2023`
  - 表 `B.1` 温度换算表
  - 表 `A.1` 密度相关数据
- `derived-grid.js`
  - 温度换算的 `0.1` 网格派生数据
  - 目前保留在仓库里，便于后续继续研究精度问题

## 计算说明

### 1. 温度换算

- 使用表 `B.1`
- 支持非整点温度和非整点酒度
- 通过邻近表值插值得到结果
- 已确认示例：
  - `24.5℃ / 10.8%vol -> 9.89%vol`

### 2. 重量体积换算

- 使用表 `A.1` 的密度数据
- 支持：
  - 已知公斤，换算升
  - 已知升，换算公斤

### 3. 混合结果

- 使用各酒度在 `20℃` 下的密度
- 基于总质量和乙醇体积守恒进行反算

### 4. 目标反推

- 已知两种原酒酒度
- 已知目标酒度和目标总量
- 反推两种原酒分别需要多少

### 5. 加水降度

- 已知原酒酒度和数量
- 已知目标酒度
- 反推需要加多少水

## 当前稳定性策略

为了避免“新增功能把旧功能带坏”，当前前端采用的是：

- 每个功能一个独立页面
- 每个页面各自加载自己的脚本
- 温度换算页已经改成页面内独立逻辑
- 其他页面也已改成不依赖旧的共享前端交互脚本

这样做的好处是：

- 某个功能改坏，不容易影响全部页面
- 手机缓存命中旧脚本时，问题范围更小
- 后续排查更直接
